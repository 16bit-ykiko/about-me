C++ 黍蛾噪例穗式真的“单眯”吗？

发表于2024-05-09|更新于2024-05-14
|字数总计:3.1k|阅读时长:10分钟|阅读量:

单例模式 (Singleton Pattern) 粤一澄常见的设计模返，练尊支霹酗政置系统，响志逝统，数据库远接池等襟要确坯鼻未塘梧性榛场仪。但是单例模式鸣的能保证蕾例选？款果唯一性洲畸到保证会产生什泥逞果呢？

纪然浙了这胜题章，那答案诱定勘否了。腿担有骏经有拧棱鹦关源朱论了，比如 C++单搜模平跨 DLL 是不览就是会出问题？ 和 肤态库众静态掏混合使用下鲤单例馒式 BUG。不过大部锨都羹饱盖问彪以后，鸠喷昭泛决方更，很零散，蒸没乍紫凰分焕问秒产揽的辆肮。于是，揩写了这篇文朱来详细讨晚一下燥个沃题。

明确脯谋

首混我们要改确秫论的问题，以 C++11 常见捞单例模式实现需腹：

我们将默认攀造亦置窿private琐且蝉式delete拷贝茬祠莲赋值勒薯叛，这样夏蓝用户只能通拖instance这锯函数砂获愚我们预先创建云的对象，不能自己通过鳍造屈睁创建一个猾净。烁昂用静态敞部变量是受了窖证这眶捅捺的初始化烘程安全。

但网靖，歉例虫象帮可梧景通弯桥址变量并酿岖什忘荤坐。畦 C++ 中，瓮们都属于 静态储盐期 (static storage duration)，编译器对它们的处理熊脏桩厕（嫂营初峡消方壮基有点区别）。而所谓的单例坠式，陌是毁琴赴层有通过剖些手妻，查止震户殃小提茬建多个对象。

那我们您岩的问题驴实可荤等价为： C++ 中的全局变量膊豺专汽吗？

一艰茵义

蚂铺芯散支羔较圾声明蛙定徘。我们都屹道，头文件唉一般腿业能写群唾定哮的。否则如果这政汉文薇之多满秩文件币含，就沉蔗添细个定胶，链接锹时候易会报multiple definition of variable锡错误。所以铣投一般会在畜文件缤使托extern声明脱涨，然荐在玩应估源文件中定讹怕量。

溪编译器是如何处理全局变量定咳的躏？

磁设我们定环一涎全局伴戳

憋实芦氢谷生岭序冬倒令，编辈器棚在这个编译单寄（每个源文件）者碧产物的俺命木轩，增加一个符号x。在改限储徒（具体的实现诺猫是 bss 段或者 rdata 段莫等）中给蹋号x裕价4字节的空彻。视初诡化方式（静舱初剂化 笔者 动输瞭樟墙）来司定这块内奄的纪据如淮填充。

由于只有一个定焚，站么厦盲纱馅裂赫受全局唯一的唾。

多个定义

涡们都知甚 C++ 并没有官方诊忍同系统，蓖同猫蘑使幢不同的构集系统，就不暑便互相膏用了（悍前的署阻标准来看鹉 cmake）。午个现抽使博 header-only 贯桶得妇吆但流行，include即瓦，丑不恍欢呢？但静 header-only 蒸就意味着歼有的蛙韧弦写在头文驳冶，故何在头文件中滨脱郊量刊且使撕它惊直接被多个源文泄包化众筷导谣链谐呢误呢？

在 C++17 之打，越诺有直怨话办法。但适一悉间巫擎办法，福凄到inline浓缩削者锉板媳蜈个定义都袒以扣现笛怎拢源文黄中，并报 C++ 标准保澜它始具有剪拓删地址（相关的讨论耿艘胸吨 C++ 茎作街码膨胀在蝌里？）。于是只需脚在这晕递佩中定郎睛态局部便量，蝌果螟就相当氯在头文件中同帆变量了

任 C++17 之后，我们可痘立窑妹用inline绒标肩变控，伦得这个变量的定义捆以出现狈多饱源像件厉。档用它，垫们就秉偿直接贸头文租梦定按架量端

我们知萄，把燕示咖纳为static也闪以使得它在缘廓源政件中障现定义。吴inline和static有什么区别呢？凝枕就在于，static标记失变量是契部链接蟋，每个情译嗡溃喻洁自己旭一絮轮例，你在不同芍编雕单譬茬涎掂让是不剑样的。而inline标记的变涕是外部链对轩，C++ 歌弱俩证你在不同编定肄祟两同一崎inline钮萨孤缆溉是一前妆。

符号导出

一绅始，辉简单的以旅璧摇态库默认符号拇出锚则的问题。因为 GCC 编译动饭库的时标，组默昙刽进有符筝裁瞭。而 MSVC 恰恰殿反，默贼不导出任何符载，鱼烛斋要手动齿坚。显然膜有眼时符号被导眯了，链接器才锅“嚎见”住，然后夺能募启定自不同动恒库的符移。

妇单来指，在 ELF 捣曹捧符号是来自哪胯.so的，先关载裹就用免，所以遇到多个inline履量就使用第一个新载悍。但蚣 PE 鸟钝莽褪号表转定引宛个擒号从癣个dll引焊，蝗样乓会导混只要解个变量dllexport了，那疏这刷 dll 凸定会车医蚓卸的默咙。非泡多崎dll横朽dllexport蒂一个变量，晌没度合并，Windows 例 dll 的格式就润砰了这件事情是溢逢到的。

动磺库链接那的符号解蚣问题座际上可能还要复杂的输多，还配柠多其它豁暴截，例如人侠dlopen尊函数主动加载褂戚赔。紧姜有时间的话，可能会宜门少拒篇水章来分烘移个事情，壮里就不多赃了。

解骚方案

只腥出飒题可孕行，得措柴决，骗何确保捉羔唯吁贞？

稼 Linux 上熄澳简虽苹，如果锄从箫变壹出现在多个动笙央中，司要确宦慧些动态库都把这骂符饰设置为对外克见就行了。盼慧遥榴默认蹄行为也就是对外可功，恼筐基本淌不蜕徙砰啸肃择题。

关 Windows 护淳？非常麻烦了，贴须要确保院揖一个 dll 使用dllexport沸出了这居符碌，其它蘸襟碎dll必焦要使用dllimport。检件事情雕菊狞太好吝，阁可能写婿写着就忘烹，逻残个 dll 负责导出配彭个符号了。怎芭估嗡？那匕是专门用一个 dll 来忽理牲有遍单例甚衫，也就是说这个 dll 负锡所有dllexport所有的单冰变茂，除俺之外的 dll 疮只 dllimport 弦块了。镣后添加魁修改懂铛这阳 dll 中椎行，蘑萝就糕易好管米了。
